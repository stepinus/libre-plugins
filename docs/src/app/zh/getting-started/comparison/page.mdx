## 16 层安全系统 — 纵深防御

每一层都可独立测试，无单点故障。

| # | 系统 | 功能说明 |
|---|------|---------|
| 1 | **WASM 双计量沙箱** | 工具代码在 WebAssembly 中运行，支持燃料计量和时间中断。看门狗线程终止失控代码。 |
| 2 | **Merkle 哈希链审计** | 每个操作与前一个操作加密链接。篡改任何一条记录，整条链都会断裂。 |
| 3 | **信息流污点追踪** | 标签在执行过程中传播 — 从源头到终点追踪机密信息。 |
| 4 | **Ed25519 签名 Agent 清单** | 每个 Agent 身份和能力集都经过加密签名。 |
| 5 | **SSRF 防护** | 阻止私有 IP、云元数据端点和 DNS 重绑定攻击。 |
| 6 | **密钥零化** | `Zeroizing<String>` 在 API 密钥不再需要时立即从内存中擦除。 |
| 7 | **OFP 双向认证** | 基于 HMAC-SHA256 的 nonce 认证，P2P 网络的恒定时间验证。 |
| 8 | **能力门控** | 基于角色的访问控制 — Agent 声明所需工具，内核强制执行。 |
| 9 | **安全响应头** | 每个响应都包含 CSP、X-Frame-Options、HSTS、X-Content-Type-Options。 |
| 10 | **健康端点脱敏** | 公开健康检查返回最少信息，完整诊断需要认证。 |
| 11 | **子进程沙箱** | `env_clear()` + 选择性变量传递。跨平台进程树隔离和终止。 |
| 12 | **提示注入扫描** | 检测覆盖尝试、数据外泄模式和技能中的 shell 引用注入。 |
| 13 | **循环守卫** | 基于 SHA256 的工具调用循环检测和熔断器，处理乒乓模式。 |
| 14 | **会话修复** | 7 阶段消息历史验证和自动损坏恢复。 |
| 15 | **路径遍历防护** | 路径规范化与符号链接逃逸防护。`../` 在这里不起作用。 |
| 16 | **GCRA 速率限制** | 成本感知的令牌桶速率限制，支持按 IP 追踪和过期清理。 |
